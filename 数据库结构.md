## 2. 主要节点结构

### 2.1 路线、成本与风险

| 标签 | 数量 | 用途 | 关键字段 |
|---|---:|---|---|
| `RouteSegment` | 594 | 可计算的有向运输边 | `segmentId`, `fromNodeName`, `toNodeName`, `mode`, `distance_km`, `estimated_cost_usd`, `estimated_time_days`, `riskScore`, `costScore`, `costRiskScore`, `risk_breakdown` |
| `RouteCostObservation` | 586 | 路段成本观测 | 成本值、观测时间、数据源 |
| `RouteDelayObservation` | 628 | 延误与时效观测 | 延误值、观测时间、数据源 |
| `RiskFactor` | 612 | 路段或实体的风险因子 | `risk_factor_id`, `risk_type`, `risk_score`, `confidence_score`, `source` |
| `RiskEvent` | 20 | 港口拥堵、罢工、冲突、运力短缺等事件 | `eventId`, `eventType`, `severity`, `affectedRegion`, `startDate`, `endDate` |
| `Route` | 6 | 预定义完整路线 | `route_id`, `name`, `total_risk_score`, `risk_breakdown` |
| `RouteRecommendation` | 15 | 历史预计算推荐结果 | `recommendationId`, `originName`, `destinationName`, `rank`, `pathNodeNames` |
| `RiskMatrixCategory` | 4 | 成本-风险矩阵分类 | 低成本/高成本、低风险/高风险 |

核心路段结构：

```text
(fromNode)<-[:FROM_NODE]-(segment:RouteSegment)-[:TO_NODE]->(toNode)
(segment)-[:USES_MODE]->(mode:TransportMode)
(segment)-[:HAS_COST_OBSERVATION]->(cost:RouteCostObservation)
(segment)-[:HAS_DELAY_OBSERVATION]->(delay:RouteDelayObservation)
(segment)-[:HAS_RISK]->(risk:RiskFactor)
(segment)-[:CLASSIFIED_AS]->(matrix:RiskMatrixCategory)
(route:Route)-[:HAS_SEGMENT]->(segment)
```

## 5. FastAPI 对外 HTTP 接口

所有业务接口目前都是只读 `GET`。Swagger 地址：`/docs`。

### 5.1 服务与健康检查

| 接口 | 用途 |
|---|---|
| `GET /` | 服务基本信息 |
| `GET /health` | FastAPI 进程和数据库配置状态 |
| `GET /health/aura` | AuraDB 连通性检查 |

### 5.2 图谱概览

| 接口 | 主要参数 | 用途 |
|---|---|---|
| `GET /api/graph/summary` | 无 | 标签和关系数量统计 |
| `GET /api/supply-chain/routes` | `limit` | 返回原始路段，主要用于调试 |
| `GET /api/risk/overview` | `limit` | 国家、港口和路段风险概览 |

### 5.3 成本与风险

| 接口 | 主要参数 | 用途 |
|---|---|---|
| `GET /api/risk/segments` | `limit`, `minimum_risk` | 按综合风险排名路段 |
| `GET /api/cost/segments` | `order`, `limit` | 按成本升序或降序排名路段 |

### 5.4 路径规划

| 接口 | 主要参数 | 用途 |
|---|---|---|
| `GET /api/suppliers` | `search`, `limit` | 供应商下拉列表 |
| `GET /api/cities` | `search`, `limit` | 起终点城市/节点下拉列表 |
| `GET /api/routes/nodes` | `search`, `limit` | 返回可传给最短路算法的 `node_id` |
| `GET /api/routes/recommend` | `supplier`, `origin`, `destination`, `limit`, `risk_weight`, `max_hops` | **前端主接口**：按供应商+起点+终点返回多条完整路线 |
| `GET /api/routes/optimize` | `origin_id`, `destination_id`, `objective`, `risk_weight` | 单条最低成本、最低风险或平衡路径 |
| `GET /api/routes/recommendations` | `objective`, `risk_weight`, `limit` | 对数据库中预定义的 `Route` 排名 |

前端主调用示例：

```http
GET /api/routes/recommend?supplier=CATL&origin=Shanghai&destination=Hamburg&limit=5&risk_weight=0.5
```

响应中每条路线包含：

```text
id, name, riskScore, cost, durationDays, distanceKm, tags, riskFactors, legs
```

`legs.from` 和 `legs.to` 包含 `name`, `city`, `country`, `lat`, `lng`, `coordinateSource`，可直接用于前端地图画线。

## 6. 当前缺口与建议

1. **AIS 尚未形成持续观测数据集**：当前只有 1 艘示例船、1 条船位和 1 个运单。
2. **苏伊士运河未配置**：如果需要监测苏伊士，需在 `ais_observation_targets.json` 新增运河南北入口或运河区域边界框。
3. **需要常驻 worker**：Render 当前 FastAPI Web Service 不会自动运行 AIS WebSocket 订阅脚本。
4. **建议新增 AIS HTTP 查询接口**：当前 FastAPI 还没有 `/api/vessels` 或 `/api/ais/observations`，前端无法直接查询 AIS 数据。
5. **坐标完整性**：路径 API 会用同城市或图邻居估算缺失坐标，`coordinateSource` 不是 `database` 时不应当作精确地理定位。

## 7. 建议的 AIS 风险评估链路

```text
AISStream PositionReport / ShipStaticData
  -> Vessel + VesselObservation
  -> 匹配最近 Port / Route / Shipment
  -> 计算船速过低、锚泊密度、航向异常、目的地偏离、吃水异常
  -> 更新 transport_delay_risk / port_congestion_risk / route_reliability_risk
  -> 重算 RouteSegment.riskScore
  -> /api/routes/recommend 返回最新路线风险
```

不指定 MMSI 时，可按港区边界框聚合船舶数量、平均速度、低速/锚泊船比例和航向分布，得到港区级拥堵与延误风险；指定 MMSI 时，再进一步形成运单级跟踪。
